Demonstrates the use of a message dialog with custom commands, command callbacks, and a default command. A message will be displayed indicating which command was invoked on the dialog.

In this scenario, 'Install updates' is selected as the default choice.



Known bluesky issues with all scenarios in this sample:
  • R2 bug: enter/exit don't work with default/cancel
  • R2 bug: keyboard focus in messagedialog can go to underlying controls
  • R2 bug: in8 inconsistent on whether it's "id" or "index" for defaultCommandIndex
